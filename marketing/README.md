## Platform-Specific Notes

### X/Twitter
- **Best time to post**: 9-11am EST or 6-8pm Nairobi (EAT)
- **Thread length**: 6 tweets (don't exceed — engagement drops after 8)
- **Hashtags**: Use #BuildOnCelo #MiniPay #Web3 #DeFi (don't overtag)
- **Images**: Every tweet should have visual content

### Farcaster
- **Channels**: /celo (main), /miniapps (distribution), /dev (technical)
- **Embeds**: Post mini app URL directly — it renders in-feed
- **Tone**: More technical, builder-focused than Twitter
- **Engagement**: Reply fast (first hour is critical)

### Celo Forum
- **Format**: Professional, proposal-style
- **Sections**: Executive Summary, Technical Architecture, Use Cases
- **Audience**: Celo foundation, ecosystem partners, investors
- **Update policy**: Edit post to add contract addresses post-deployment

### MiniPay Forum
- **Format**: Formal submission with technical requirements
- **Key sections**: MiniPay compatibility, user journey, screenshots
- **SLA commitment**: 24h critical bug fix promise
- **Metrics**: Be realistic with 30/90-day goals

### Moltbook
- **Format**: Concise hackathon submission
- **Focus**: Innovation, technical execution, impact
- **Include**: Demo link, metrics, future roadmap
